# Go 排序算法实现与对比

本项目使用 Go 语言实现了多种常见的排序算法，并对它们的特性进行了分析和比较。

## 算法列表

- 冒泡排序 (Bubble Sort)
- 选择排序 (Selection Sort)
- 插入排序 (Insertion Sort)
- 希尔排序 (Shell Sort)
- 归并排序 (Merge Sort)
- 快速排序 (Quick Sort)
- 堆排序 (Heap Sort)

## 性能与特性对比

| 排序算法 (Sorting Algorithm) | 平均时间复杂度 (Average Time) | 空间复杂度 (Space) | 稳定性 (Stability) | 适用场景 (Use Cases) |
| :--- | :---: | :---: | :---: | :--- |
| **冒泡排序 (Bubble Sort)** | O(n²) | O(1) | **稳定 (Stable)** | - 数据量极小的情况<br>- 用于教学或理解排序概念 |
| **选择排序 (Selection Sort)** | O(n²) | O(1) | 不稳定 (Unstable) | - 数据量小<br>- 关注空间复杂度，要求原地排序 |
| **插入排序 (Insertion Sort)** | O(n²) | O(1) | **稳定 (Stable)** | - 数据量小<br>- 数组基本有序时效率很高 |
| **希尔排序 (Shell Sort)** | O(n log n) | O(1) | 不稳定 (Unstable) | - 插入排序的改进版，适用于中等规模数据 |
| **归并排序 (Merge Sort)** | O(n log n) | O(n) | **稳定 (Stable)** | - 追求稳定性能，不在乎额外空间<br>- 需要稳定排序的场景 |
| **快速排序 (Quick Sort)** | O(n log n) | O(log n) | 不稳定 (Unstable) | - 大多数情况下的首选，平均性能最好<br>- 对性能要求高，不要求稳定性 |
| **堆排序 (Heap Sort)** | O(n log n) | O(1) | 不稳定 (Unstable) | - 需要原地排序且时间复杂度稳定<br>- 寻找 Top K 问题 |

---

### 概念解释

- **时间复杂度 (Time Complexity)**: 描述算法运行时间随输入规模增长而增长的量级。
- **空间复杂度 (Space Complexity)**: 描述算法在运行过程中临时占用存储空间大小的量度。
- **稳定性 (Stability)**: 如果两个相等的元素在排序前后的相对位置保持不变，则称该排序算法是稳定的。
- **原地排序 (In-place Sort)**: 指的是空间复杂度为 O(1) 的排序算法。

### 如何选择？

- **数据量小**时，可以选择 **插入排序**、**冒泡排序** 或 **选择排序**，它们实现简单。如果数据基本有序，**插入排序** 表现最佳。
- **数据量大**时：
    - 如果关注**平均性能**，**快速排序** 是最佳选择。
    - 如果需要**稳定的排序结果**（例如，按多个字段排序），**归并排序** 是不二之选。
    - 如果**空间资源受限**，要求原地排序，且希望有稳定的 O(n log n) 性能，**堆排序** 是一个很好的选择。
